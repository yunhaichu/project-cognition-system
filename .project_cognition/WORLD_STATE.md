# WORLD_STATE.md

## 1. Project Nature
This repository implements a low-drift, auditable project cognition system for AI agents. It is not ordinary RAG, not an infinite memory file, and not a mechanism for injecting all conversation history into every prompt.

## 2. Core User Intent
Keep project understanding stable while minimizing context cost. User utterances and real tool results have the highest evidence weight. Agent final answers are logs, not core facts.

## 3. Current Project Status
MVP implementation using local files, Markdown, JSONL, and Python standard-library scripts.

## 4. Stable Architecture Principles
Each project owns its own `.project_cognition/` state. User profile is global and agent-specific. Core state is rebuilt through controlled scripts with confidence scoring and conflict detection.

## 5. Non-Negotiables
- Do not bulk-inject all historical conversation.
- Do not let memory grow into a giant prompt.
- Do not treat agent output as high-confidence memory.
- Do not promote low-confidence or evidence-free claims into core state.
- Do not silently overwrite unresolved conflicts.

## 6. High-Risk Misunderstandings
Do not treat this system as a vector database wrapper. Its purpose is cognition governance: evidence weighting, conflict handling, reviewable updates, and compact world-state generation.

## 7. Current Strategy
Keep the MVP simple: raw evidence -> candidates -> scoring -> conflicts -> compact world state -> proposed updates -> review.

## 8. Drift Check
Before acting, check whether the task violates user evidence, treats summaries as facts, expands scope, ignores real files or tool output, or needs targeted source lookup.

## 9. Update Rules
`WORLD_STATE.md` is generated by controlled scripts. New cognition should enter proposals or confidence tables first, then be rebuilt into core state after review.
